# JobTrack 版本计划

## 1. 计划原则

- 每个版本只聚焦一个可验收主题，具体范围在开发前再次人工确认。
- 以下能力均为计划；只有代码、测试、文档和人工验收全部完成后才能标记为已完成。
- 后续版本可以根据反馈调整，但不得擅自合并、删除或提前完成约定版本。
- 本路线不自动授权引入依赖或开始开发；每个版本都必须单独制定计划并等待人工确认。
- 每个版本都要更新 `CHANGELOG.md`，并包含明确目标、验收标准和当前状态。

## 2. 版本路线

### v0.1.0：项目初始化与开发规则

**目标**

建立 JobTrack 的项目定位、业务边界、长期开发规则、完整版本路线和人工验收基线，为后续开发提供一致依据。

**验收标准**

- 九个约定文件齐全且内容一致；
- README 明确当前没有 Spring Boot 骨架且暂时不可启动；
- 未创建 Java 业务代码、Controller、Service、Mapper、`pom.xml`、生产依赖或数据库连接；
- 完成文件范围、敏感信息、本机绝对路径、忽略规则和 Git 状态检查；
- 版本路线完整列出 v0.1.0 至 v1.3.0 的所有约定版本。

**当前状态**

已发布

### v0.2.0：Spring Boot 项目骨架

**目标**

创建最小、可运行、可测试的 Spring Boot 单体工程骨架，建立基础目录、构建和环境配置方式。

**验收标准**

- Java、Spring Boot、Maven 及最小依赖版本经人工确认；
- 应用能够在不连接生产资源的本地环境启动；
- 基础目录结构和配置分层清晰，示例配置可安全提交；
- 最小上下文加载测试通过；
- README 提供经过验证的构建、测试和启动命令。

**当前状态**

已发布

### v0.3.0：MySQL 表结构与用户上下文

**目标**

设计并建立 MVP 所需的 MySQL 表结构与可信用户上下文，为后续业务模块提供数据归属基础。

**验收标准**

- 表、字段、索引、约束和初始化方案经人工确认；
- 核心业务数据具备明确的用户归属字段或归属链；
- 本地配置不包含真实凭据，示例配置只使用占位值；
- MyBatis 参数绑定和用户上下文传递方式经过验证；
- 初始化、回滚和空库验证步骤有文档记录。

进行中

## 3. 状态维护

版本状态只使用“计划”“进行中”“已验收”“已发布”。人工确认开始开发后才能改为“进行中”；通过验收后才能改为“已验收”；合并并完成发布动作后才能改为“已发布”。当前 v0.1.0 和 v0.2.0 处于“已发布”，v0.3.0 处于“进行中”，其余所有版本均为“计划”。
