# JobTrack 产品范围

## 1. 产品目标

JobTrack 计划帮助个人求职者集中记录和整理本人求职过程，减少信息分散、跟进遗漏和复盘困难。产品只管理用户主动录入或合法导入的个人求职数据，不替代招聘平台。

## 2. 目标用户与核心场景

目标用户是需要同时跟进多个求职机会的个人求职者。核心场景包括：

- 记录关注的公司和岗位；
- 记录岗位投递状态与关键日期；
- 记录面试安排、结果和复盘；
- 记录与招聘方的沟通摘要；
- 按个人数据查看后续统计和进度概览。

以上均为产品总体规划；v0.1.0 不实现这些业务能力。

## 3. 核心业务对象

| 对象 | 含义 | 归属规则 |
| --- | --- | --- |
| User | 使用 JobTrack 的个人用户 | 用户只能访问自己的身份和数据 |
| Company | 用户关注或投递相关的公司 | 直接归属于 User |
| Job | 某公司的具体岗位 | 归属于 User，并关联 Company |
| Application | 用户对某岗位的一次投递 | 归属于 User，并关联 Job |
| Interview | 某次投递下的面试记录 | 通过 Application 归属于 User |
| Communication | 某次投递下的沟通记录 | 通过 Application 归属于 User |
| Statistic | 基于上述数据计算的个人统计 | 只能聚合当前 User 的数据 |

对象字段、状态枚举和数据库结构需要在对应实现版本中另行确认。

## 4. 产品范围内

- 单个用户管理自己的求职过程数据；
- 公司、岗位、投递、面试和沟通的基础记录与关联；
- 明确且可追踪的投递状态流转；
- 个人维度的查询、筛选与统计；
- 用户间严格的数据隔离；
- 必要的数据校验、错误反馈和审计日志。

## 5. 产品范围外

- 招聘方、猎头、企业组织和多租户协作后台；
- 职位聚合、网页抓取、自动申请或批量投递；
- 简历制作、公开人才库、在线聊天和商业撮合；
- 薪酬、员工入职、考勤、绩效等 HR 系统能力；
- 社交网络、推荐广告和付费交易；
- 未经明确版本规划的数据智能决策或自动化代理。

## 6. 业务边界原则

- 服务端从可信认证上下文获取当前用户，不能信任客户端提交的 `userId`。
- 所有查询、修改、删除、统计和导出必须限定当前用户的数据。
- 不采集实现目标所不需要的个人敏感信息。
- 计划功能只有完成实现、测试和人工验收后，才能标记为已完成。
- 新能力若改变目标用户、核心对象或范围外清单，必须先更新本文档并人工确认。

## 7. v0.1.0 目标与验收标准

### 目标

建立产品定位、范围、业务对象和边界基线，为后续版本提供统一依据。

### 验收标准

- 清晰区分总体规划与 v0.1.0 已交付范围；
- 明确目标用户、核心场景、业务对象、范围内和范围外能力；
- 明确 `userId` 数据隔离和敏感信息原则；
- 与 README、业务模型、版本计划和开发规则没有冲突；
- 不包含 Java 代码、构建配置或数据库连接实现。
